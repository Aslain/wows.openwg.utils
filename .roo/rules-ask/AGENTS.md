# AGENTS.md

This file provides guidance to agents when working with code in this repository.

- The installer API docs are generated by concatenating each module’s `.iss` stub onto the shared script during configure time ([src/CMakeLists.txt](src/CMakeLists.txt:65)), so reference the module-specific `api_*.iss` files for authoritative call signatures instead of README snippets.
- End-user examples intentionally live inside the Inno fragments (e.g. JSON usage at [src/json/test_json.iss](src/json/test_json.iss:21)) rather than markdown; quote those scripts when documenting behaviors.
- The test installer flow is documented only through the harness in [src/test/test_prolog.iss](src/test/test_prolog.iss:59) and [src/test/test_epilog.iss](src/test/test_epilog.iss:3); surface both when describing how results land in `test_result.json`.
- UI authors should note the drag-and-drop HTML reporter in [src/test/test_report.html](src/test/test_report.html:45); it is the canonical description of the JSON result schema and is not mentioned elsewhere.
